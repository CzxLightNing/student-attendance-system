## 1. 项目概述
构建一个基于 Django 的学生考勤 Web 系统，支持用户注册与登录、教师生成签到码、学生输入签到码签到，并提供管理员、教师、学生三种角色各自的管理与查看功能。前端采用响应式设计，兼容桌面端、平板端和移动端等多终端设备，使用原生 JavaScript 实现 Bootstrap 5 样式，不依赖任何前端 JS 框架。

## 2. 技术栈
- 后端：Python 3.10+, Django 4.2 LTS
- 数据库：SQLite（开发环境）；数据库设计需考虑后续迁移至 MySQL 的兼容性，模型字段、索引、约束等应遵循 MySQL 规范
- 前端：原生 HTML5、CSS3、JavaScript（ES6）；Bootstrap 5 响应式栅格系统，兼容桌面端（≥1200px）、平板端（≥768px）、移动端（<768px）
- 认证：Django 内置认证系统，扩展自定义用户模型
- 密码哈希：使用 `argon2-cffi` 库进行哈希加盐后存储，严禁明文存储密码；配置为 Django 默认密码哈希算法
- 配置管理：使用 `python-dotenv` 从 `.env` 文件加载敏感配置（数据库账号密码、SECRET_KEY 等）
- 导出：使用 `openpyxl` 生成 Excel 考勤表单
- 时间同步：使用 `ntplib` 库从 `ntp.aliyun.com` 获取 NTP 服务器时间戳，确保系统时间准确一致
- 安全防护：使用 `django-axes` 实现登录失败次数限制与账号临时锁定，防止暴力破解攻击
- 其他：Django messages framework 用于用户反馈

## 4. 功能需求详解

### 4.1 用户认证与基础数据

**用户模型：** 自定义 User 模型（继承 AbstractUser），增加字段：
- `role`：可选 `student`, `teacher`, `admin`
- `student_id`（学号，仅学生角色）、`teacher_id`（工号，仅教师角色）可为空
- 关联班级：`student_class`（ForeignKey 到 Class，仅学生），`managed_classes`（ManyToMany 到 Class，仅教师）

**班级模型：** `Class` 模型包含字段：
- `name`：班级名称（唯一）
- `grade`：年级
- `created_at`：创建时间

**用户注册：** 系统提供用户注册页面，访问者可自行注册账号（选择学生或教师角色），填写必要信息后提交，由管理员在后台审核/激活。

**管理员用户管理：**
- 管理员可批量导入数据（上传 CSV 文件）：支持批量导入班级、批量导入学生（含所属班级）、批量导入教师账号
- 管理员也可逐个手动添加学生、教师账号和创建班级

**密码管理策略：**
- **所有用户密码均使用 `argon2-cffi` 进行哈希加盐后存入数据库，严禁以任何形式存储明文密码**
- 管理员创建或批量导入学生/教师账号时，初始密码统一为 `12345678`，保存时需经过 `argon2-cffi` 哈希处理
- 管理员可在用户管理页面重置任意学生或教师账号密码，重置后密码恢复为初始密码 `12345678`，同样需哈希后存储
- 学生和教师登录后可在各自后台的"修改密码"页面自行修改密码，需验证原密码后设置新密码，新密码同样经 `argon2-cffi` 哈希存储

### 4.2 签到码生成（教师）
- 教师登录后进入"我的班级"列表，选择一个班级进入签到管理
- 可生成一个**4位数字签到码**（随机生成，当前班级唯一有效）
- 签到码有过期时间：教师可设定有效时长（如5分钟、10分钟、自定义），或默认10分钟
- 同时只能有一个有效签到码，生成新码时旧码自动失效
- 教师可手动失效当前签到码
- **签到实时监控**：教师生成签到码后，页面分为"已签到"和"未签到"两个区域，系统每隔 3~5 秒自动轮询获取最新签到数据，无需手动刷新页面即可实时看到哪些学生已签到、哪些学生尚未签到

签到码模型：
```python
class AttendanceCode(models.Model):
    class_group = models.ForeignKey(Class, on_delete=models.CASCADE)
    code = models.CharField(max_length=4)
    created_by = models.ForeignKey(User, on_delete=models.CASCADE)
    created_at = models.DateTimeField(auto_now_add=True)
    expires_at = models.DateTimeField()
    course_start_time = models.DateTimeField(null=True, blank=True)
    is_active = models.BooleanField(default=True)
```
### 4.3 学生签到
- 学生登录后主页显示签到码输入框
- 输入4位数字，提交后系统校验： 
  - 签到码是否存在且有效（未过期、未失效）
  - 学生是否属于该签到码对应的班级
  - 该学生在本次签到码有效期内是否已签到（防止重复）
- 成功则记录签到，失败显示具体错误（无效码、非本班、已签到等）
- 签到记录模型包含：`student`, `attendance_code`, `timestamp`

### 4.4 考勤详情查看
- 教师/管理员可查看每一次生成的签到码对应的签到记录（哪个学生、签到时间、是否迟到等）。
- 签到码列表可展开显示已签到/未签到学生。
- 管理员可查看所有班级，教师只能查看自己管理的班级。
- 教师可在班级管理页面查看所管理班级的完整学生名单（学号、姓名等），但无权修改学生信息。

### 4.5 导出考勤表单
- 在考勤详情页面提供“导出Excel”按钮
- 导出当前签到码下的签到记录，包含字段：学号、姓名、签到时间、状态（正常/迟到/缺勤）
- 使用 `openpyxl` 生成 `.xlsx` 文件并下载
- 支持按日期范围导出某个班级的全部签到汇总

### 4.6 管理面板（管理员专用）
- 班级管理：增删改班级；支持通过 CSV 文件批量导入班级
- 用户管理：增删改教师、学生，按班级筛选学生；支持通过 CSV 文件批量导入教师和批量导入学生；支持重置任意学生或教师账号密码（重置为初始密码 `12345678`）；支持审核/激活用户自行注册的账号
- 考勤总览：查看所有班级的签到统计
- 操作日志（可选）：记录管理员重要操作

**CSV 批量导入说明：**
- 批量导入页面必须提供 **CSV 示例文件下载**，用户可下载标准格式的模板文件后按格式填写数据再上传导入
- **班级 CSV 示例**（`classes_template.csv`）：`班级名称,年级` → `一年级一班,一年级`
- **教师 CSV 示例**（`teachers_template.csv`）：`用户名,工号,姓名,邮箱,管理的班级名称` → `teacher01,T001,张老师,zhang@school.com,一年级一班`
- **学生 CSV 示例**（`students_template.csv`）：`用户名,学号,姓名,所属班级名称` → `student01,S2026001,李明,一年级一班`

## 5. 数据库模型设计

### 5.1 实体关系概览
系统包含 4 个核心实体，关系如下：
- **班级（Class）** — 1:N — **用户（CustomUser）**（一个班级有多个学生）
- **用户（CustomUser）** — N:M — **班级（Class）**（一个教师可管理多个班级）
- **用户（CustomUser）** — 1:N — **签到码（AttendanceCode）**（教师创建签到码）
- **班级（Class）** — 1:N — **签到码（AttendanceCode）**（签到码属于某个班级）
- **签到码（AttendanceCode）** — 1:N — **签到记录（AttendanceRecord）**（一次签到码对应多条签到记录）
- **用户（CustomUser）** — 1:N — **签到记录（AttendanceRecord）**（学生有多条签到记录）

#### 5.3.2 常用查询示例（SELECT）
```sql
-- 1. 查询某班级当前有效的签到码
SELECT * FROM attendance_attendancecode
WHERE class_group_id = 1
  AND is_active = 1
  AND expires_at > NOW()
ORDER BY created_at DESC
LIMIT 1;

-- 2. 查询某次签到码对应的签到记录（含学生姓名）
SELECT ar.id, ar.timestamp, ar.status,
       u.username, u.first_name, u.last_name, u.student_id
FROM attendance_attendancerecord ar
JOIN accounts_customuser u ON ar.student_id = u.id
WHERE ar.attendance_code_id = 1
ORDER BY ar.timestamp;

-- 3. 查询某班级未签到的学生名单（签到码下缺勤学生）
SELECT u.id, u.username, u.first_name, u.last_name, u.student_id
FROM accounts_customuser u
WHERE u.role = 'student'
  AND u.student_class_id = 1
  AND u.id NOT IN (
    SELECT student_id FROM attendance_attendancerecord
    WHERE attendance_code_id = 1
  );

-- 4. 查询某学生在某日期范围内的签到历史
SELECT ar.timestamp, ar.status,
       ac.code, cg.name AS class_name
FROM attendance_attendancerecord ar
JOIN attendance_attendancecode ac ON ar.attendance_code_id = ac.id
JOIN class_group cg ON ac.class_group_id = cg.id
WHERE ar.student_id = 1
  AND ar.timestamp BETWEEN '2026-01-01' AND '2026-06-30'
ORDER BY ar.timestamp DESC;

-- 5. 统计某班级所有签到码的签到率
SELECT ac.id, ac.code, ac.created_at,
       COUNT(ar.id) AS signed_count,
       (SELECT COUNT(*) FROM accounts_customuser
        WHERE student_class_id = 1 AND role = 'student') AS total_students
FROM attendance_attendancecode ac
LEFT JOIN attendance_attendancerecord ar ON ac.id = ar.attendance_code_id
WHERE ac.class_group_id = 1
GROUP BY ac.id
ORDER BY ac.created_at DESC;
```

### 5.4 MySQL 兼容性要求
> **MySQL 兼容性要求：** 虽然当前开发阶段使用 SQLite，但所有模型设计需遵循以下 MySQL 兼容规范，确保后续迁移时无需大改：
> - 所有 CharField 和 TextField 字段必须显式指定 `max_length`，避免使用无限制的 TextField
> - BooleanField 应使用 `default` 值，避免 NULL 布尔值问题
> - DateTimeField 使用 `auto_now_add` 和 `auto_now` 时需注意 MySQL 5.6.5+ 才支持多个自动时间戳列
> - 避免使用 SQLite 特有的字段类型或查询语法（如 `contains` 对 JSONField 的支持差异）
> - 外键字段显式设置 `on_delete` 行为，MySQL 对此约束更严格
> - 模型 Meta 中为关键查询字段添加 `indexes`，MySQL 中索引对性能影响显著
> - 避免使用 Django 的 `JSONField` 除非明确需要，MySQL 5.7+ 才原生支持 JSON 类型

## 7. 前端页面要求

### 7.1 模板与样式

  - **CDN 引入**：通过 Bootstrap CDN 直接引入（适用于快速开发）

### 7.2 响应式设计要求
- **必须采用响应式设计，兼容以下终端设备：**

| 终端类别 | 屏幕宽度范围 | Bootstrap 断点 | 布局策略 |
|---------|-------------|---------------|---------|
| 移动端（手机） | < 768px | `sm` / `xs` | 单列布局，导航栏折叠为汉堡菜单，表单全宽，表格改为纵向卡片式或横向滚动 |
| 平板端 | 768px ~ 1199px | `md` / `lg` | 双列或多列适中布局，侧边栏收窄，表格正常显示 |
| 桌面端 | ≥ 1200px | `xl` / `xxl` | 多列完整布局，充分利用屏幕宽度，侧边栏展开 |

- **响应式适配要点：**
  - **导航栏**：移动端使用 `navbar-expand-lg` 折叠为汉堡菜单，展开时覆盖全屏
  - **栅格系统**：所有页面布局使用 Bootstrap 栅格（`row` + `col-*`），必须为每个元素指定不同断点的列宽（如 `col-12 col-md-6 col-lg-4`）
  - **表格**：在移动端使用 `table-responsive` 类实现横向滚动，教师签到监控页面的大表格必须在移动端可正常查看
  - **表单**：输入框、按钮在移动端使用 `w-100` 全宽，桌面端恢复默认宽度
  - **签到码显示**：大号签到码在移动端应适当缩小字体，确保不超出屏幕
  - **已签到/未签到列表**：移动端上下堆叠排列，桌面端左右并排
  - **图片/图表**：使用 `img-fluid` 确保自适应缩放
  - **间距**：移动端适当减小 `padding` 和 `margin`（使用 `p-md-4 p-2` 等响应式间距类）
  - **字体**：移动端标题字号适当缩小（使用 Bootstrap 响应式排版类）
  - **view 视口**：必须设置 `<meta name="viewport" content="width=device-width, initial-scale=1.0">`

### 7.3 页面结构

## 8. 关键业务逻辑细节
- 签到码生成算法：`random.randint(1000, 9999)`，并检查与当前班级有效码不重复
- 签到校验逻辑：
  1. 根据 code 和 is_active=True 且 expires_at > now 查找 AttendanceCode
  2. 若不存在返回“签到码无效或已过期”
  3. 检查 request.user 是否属于该班级学生
  4. 检查 AttendanceRecord 中是否存在同学生同 AttendanceCode 记录，避免重复
  5. 创建记录，返回成功
- 迟到判定：如果教师设置了签到码对应的课程开始时间（可选），签到时间晚于开始时间则标记为“迟到”，否则为“正常”。若未设置开始时间，则一律“正常”。
- 缺勤学生：在考勤详情中，可通过对比班级所有学生与已签到学生得出缺勤名单。

## 13. 生产环境安全设计

> **⚠️ 安全是系统的基础保障，以下安全措施必须在代码中实现，不可省略。**

### 13.1 Django 生产环境配置

项目必须支持通过环境变量区分开发与生产环境，`settings.py` 中实现以下配置切换：

```python
# 从环境变量读取运行模式，默认开发模式
ENVIRONMENT = os.getenv('DJANGO_ENV', 'development')

if ENVIRONMENT == 'production':
    DEBUG = False
    ALLOWED_HOSTS = os.getenv('ALLOWED_HOSTS', '').split(',')
else:
    DEBUG = True
    ALLOWED_HOSTS = ['*']
```

| 配置项 | 开发环境 | 生产环境 |
|--------|---------|---------|
| `DEBUG` | `True` | `False`（严禁在生产环境开启） |
| `ALLOWED_HOSTS` | `['*']` | 从 `.env` 读取，仅允许实际域名和 IP |
| `SECRET_KEY` | 可内置默认值 | 必须从 `.env` 读取，长度 ≥ 50 字符随机字符串 |

### 13.2 HTTPS 与安全 Cookie

生产环境必须强制启用 HTTPS 及安全的 Cookie 策略：

```python
if ENVIRONMENT == 'production':
    # 强制 HTTPS 重定向
    SECURE_SSL_REDIRECT = True
    # Cookie 仅通过 HTTPS 传输
    SESSION_COOKIE_SECURE = True
    CSRF_COOKIE_SECURE = True
    # 防止 JavaScript 读取 Session Cookie
    SESSION_COOKIE_HTTPONLY = True
    # 限制跨站请求中的 Cookie 发送
    SESSION_COOKIE_SAMESITE = 'Lax'
    CSRF_COOKIE_SAMESITE = 'Lax'
    # HSTS 头部（浏览器强制 HTTPS，有效期 1 年）
    SECURE_HSTS_SECONDS = 31536000
    SECURE_HSTS_INCLUDE_SUBDOMAINS = True
    SECURE_HSTS_PRELOAD = True
```

### 13.3 安全 HTTP 响应头

通过 Django 中间件或 `django.middleware.security.SecurityMiddleware` 配置以下安全头部：

| 安全头 | 配置值 | 防护目标 |
|--------|--------|---------|
| `X-Frame-Options` | `DENY` | 防止网站被嵌入 iframe，杜绝点击劫持攻击 |
| `X-Content-Type-Options` | `nosniff` | 禁止浏览器 MIME 类型嗅探，防止恶意文件执行 |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | 控制跨域请求中 Referer 信息泄露 |
| `Content-Security-Policy` | `default-src 'self'; script-src 'self'; style-src 'self' https://cdn.jsdelivr.net;` | 限制资源加载来源，防御 XSS 攻击 |

```python
# Django settings.py 安全头配置
SECURE_BROWSER_XSS_FILTER = True
SECURE_CONTENT_TYPE_NOSNIFF = True
X_FRAME_OPTIONS = 'DENY'
```

### 13.4 登录暴力破解防护

使用 `django-axes` 库实现登录失败限制：

| 策略 | 配置 | 说明 |
|------|------|------|
| 失败次数限制 | 同一账号 5 次失败后锁定 30 分钟 | 防止针对特定账号的密码猜测 |
| IP 限制 | 同一 IP 10 次失败后锁定 15 分钟 | 防止分布式暴力破解 |
| 锁定后行为 | 返回统一错误信息，不泄露账号是否存在 | 防止用户名枚举攻击 |
| 重置锁定 | 登录成功后自动清除失败计数 | 正常用户登录不受长期影响 |

```python
# django-axes 配置
AXES_FAILURE_LIMIT = 5                    # 失败次数阈值
AXES_COOLOFF_TIME = 0.5                   # 锁定冷却时间（小时），0.5=30分钟
AXES_LOCKOUT_PARAMETERS = ['username', 'ip_address']  # 同时按用户名和IP锁定
AXES_RESET_ON_SUCCESS = True              # 登录成功后重置计数
AXES_LOCKOUT_CALLABLE = None              # 不显示特定原因
```

### 13.5 文件上传安全

CSV 导入功能必须实施以下安全校验：

| 校验项 | 规则 | 说明 |
|--------|------|------|
| 文件类型 | 仅允许 `.csv` 后缀，MIME 类型为 `text/csv` | 防止上传可执行文件或恶意脚本 |
| 文件大小 | 单个文件不超过 5 MB | 防止拒绝服务攻击（DoS） |
| 内容校验 | 检查 CSV 列数是否匹配预期格式，过滤空行 | 防止格式错误的文件导致异常 |
| 文件名安全 | 使用 `uuid` 或时间戳重命名上传文件，保留原始扩展名 | 防止路径遍历攻击和文件名冲突 |

```python
# CSV 上传安全校验示例
ALLOWED_EXTENSIONS = ['.csv']
MAX_UPLOAD_SIZE = 5 * 1024 * 1024  # 5MB

def validate_csv_upload(file):
    # 检查文件扩展名
    ext = os.path.splitext(file.name)[1].lower()
    if ext not in ALLOWED_EXTENSIONS:
        raise ValidationError('仅允许上传 .csv 格式文件')
    # 检查文件大小
    if file.size > MAX_UPLOAD_SIZE:
        raise ValidationError(f'文件大小不能超过 {MAX_UPLOAD_SIZE // (1024*1024)}MB')
    # 检查 MIME 类型
    if file.content_type != 'text/csv':
        raise ValidationError('文件类型不正确')
```

### 13.8 密码安全策略

| 策略 | 配置 | 说明 |
|------|------|------|
| 哈希算法 | `argon2-cffi`（`Argon2PasswordHasher`） | Argon2 为当前最安全的密码哈希算法，抗 GPU/ASIC 暴力破解 |
| 密码最小长度 | 6 位 | 通过 Django 内置 `MinimumLengthValidator` 校验 |
| 常见密码禁用 | 启用 `CommonPasswordValidator` | 防止使用 `123456`、`password` 等弱密码 |
| 纯数字密码禁用 | 启用 `NumericPasswordValidator` | 防止使用纯数字密码 |
| 相似度校验 | 可选启用 `UserAttributeSimilarityValidator` | 防止密码与用户名/邮箱过于相似 |
| 初始密码 | `12345678` | 首次登录后强制提示修改密码 |

```python
# Django 密码验证器配置
AUTH_PASSWORD_VALIDATORS = [
    {'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator', 'OPTIONS': {'min_length': 6}},
    {'NAME': 'django.contrib.auth.password_validation.CommonPasswordValidator'},
    {'NAME': 'django.contrib.auth.password_validation.NumericPasswordValidator'},
]
```
